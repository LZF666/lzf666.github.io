button 标签是块级元素还是内联元素？inline-block 的概念是什么？

button 是 inline-block 元素；
inline-block 的概念：
块级对象元素会单独占一行显示，并且可以设置 width,height 属性
内联对象元素前后不会产生换行，一系列 inline 元素都在一行内显示，直到该行排满，对 inline 元素设置 width,height 属性无效
既希望元素具有宽度高度特性，又具有同行特性，这个时候我们可以使用 inline-block。
display:inline-block 旁边的内联对象会被呈递在同一行内，允许空格。

【扩展】
块级元素的特点：

  • block 元素会独占一行，多个 block 元素会各自新起一行。默认情况下，block 元素宽度自动填满其父元素宽度。
  • block 元素可以设置 width,height 属性。块级元素即使设置了宽度,仍然是独占一行。
  • block 元素可以设置 margin 和 padding 属性。
  • 常用块级元素：
    address - 地址
    blockquote - 块引用
    center - 举中对齐块
    dir - 目录列表
    div - 常用块级容易，也是 css layout 的主要标签
    dl - 定义列表
    fieldset - form 控制组
    form - 交互表单
    h1 - 大标题
    h2 - 副标题
    h3 - 3 级标题
    h4 - 4 级标题
    h5 - 5 级标题
    h6 - 6 级标题
    hr - 水平分隔线
    isindex - input prompt
    menu - 菜单列表
    noframes - frames 可选内容（对于不支持 frame 的浏览器显示此区块内容）
    noscript - 可选脚本内容（对于不支持 script 的浏览器显示此内容）
    ol - 排序列表
    p - 段落
    pre - 格式化文本
    table - 表格
    ul - 非排序列表

内联元素的特点：

  • inline 元素不会独占一行，多个相邻的行内元素会排列在同一行里，直到一行排列不下，才会新换一行，其宽度随元素的内容而变化。
  • inline 元素设置 width,height 属性无效。
  • inline 元素的 margin 和 padding 属性，水平方向的 padding-left, padding-right, margin-left, margin-right 都产生边距效果；但竖直方向的 padding-top, padding-bottom, margin-top, margin-bottom 不会产生边距效果。
  • 常用内联元素：

    abbr - 缩写

    b - 粗体（不推荐）
    bdo - bidi override

    em - 强调

    i - 斜体
    img - 图片
    input - 输入框
    kbd - 定义键盘文本
    label - 表格标签
    q - 短引用
    s - 中划线（不推荐）
    samp - 定义范例计算机代码
    select - 项目选择
    small - 小字体文本
    span - 常用内联容器，定义文本内区块
    strike - 中划线
    strong - 粗体强调
    sub - 下标
    sup - 上标
    textarea - 多行文本输入框
    tt - 电传文本
    u - 下划线
    var - 定义变量

【扩展 2】
img、input 到底是行内还是块级元素？
img 是行内元素，但同时它也是行内替换元素，行内替换元素就是会根据标签属性来显示的元素。
比如 img 根据 src 属性来显示，input 根据 value 属性来显示，因此可知道 img 和 input 是行内替换元素。
特点：height/width/padding/margin 均可用。效果等于块元素。例如 img、input、textarea、select 都是替换元素，例如 input,设置上下 padding，可以撑大父元素。
行内非替换元素，例如 strong, 设置上下 padding，只是范围扩大，可是无法撑大父元素，不会影响 line-height;

positon 属性值有哪些？

position 的属性值共有四个常用的：static、relative、absolute、fixed。

  • Static
    所有元素在默认的情况下 position 属性均为 static。
  • Relative
    relative 定位，又称为相对定位，相对定位相对的是它原本在文档流中的位置而进行偏移，没有脱离文档流，top 等属性值不影响后面的元素，而添加 margin，占据文档空间，会影响后面的元素，得往后移动。
  • Absolute
    absolute 定位，也称为绝对定位，使用 absolut 定位的元素脱离文档流后，就只能根据祖先类元素(父类以上)进行定位，这个祖先类还必须是以 position 非 static 方式定位，直到 html 根元素为止。注意：祖先类的 margin 会让子类的 absolute 跟着偏移，而 padding 却不会让使用了 absolute 发生了偏移。
  • Fixed
    fixed 和 absolute 有很多共同点： 1.会改变行内元素的呈现模式，使 display 变更为 block。 2.会让元素脱离文档流，不占据空间。 3.默认会覆盖到非定位元素上。
    absolute 的”根元素“是可以被设置的，而 fixed 则其”根元素“固定为浏览器窗口。

盒模型

W3C 标准盒模型：width=content，总宽度：width+padding+margin
IE 盒模型：width=content+padding+border，总宽度：width+margin
准确来说，是问盒模型的计算方式。

什么是浮动，如何清除浮动？

浮动：元素会脱离文档流，根据设定的值向左或者向右偏移，直到碰见了父元素的边界或者另一个浮动元素的边框。
对于父元素来说，如果它的子元素全都是浮动元素，而父元素自身没有定义高度的话，那么它会失去高度。
对于普通元素来说，它感知不到浮动元素的存在，如果宽高合适的话，会占据浮动元素原有的位置。
对于文字来说，能感知到它的存在，会围绕浮动元素分布。
清除浮动就是清除浮动元素带来的不利的影响，清除浮动的三种方式：

  1. 在父元素的子元素后面添加一个空的 div，然后在上面设置 clear:both。
  2. 使用伪元素，:after，设置 clear:both 来清除浮动。
  3. 利用 BFC，创建一个独立的不干扰其它元素的块级上下文。
    以下元素可以触发 BFC：
    overflow:不为 visible 默认值，比如说是 hidden
    float 属性不为 none
    position 值为 absolute 或者 fixed

CSS 有哪些垂直居中方案

6 种垂直居中：

  1. flex 布局
    父元素固定高度，写以下三个属性
  1. absolute margin auto
    父元素相对定位并具有固定高度，子元素绝对定位，具有高度，且 top 等值全部为 0，设置 margin:auto;即可
  2. absolute translate-50%
    父元素相对定位并具有固定高度，子元素绝对定位，有自身高度和宽度，并且 top，left 都为 50%，transform:translate(-50%,-50%)
    因为是以左上角为准的。
  3. absolute margin 负值
    同方法 3 类似，父元素固定高度，子元素都固定宽高但是是不再使用 translate，而是使用 margin-left：负自身宽度的一半,margin-top:负自身高度的一半。
  4. div 伪装成 table
    父元素固定高度，display:table
    下面还有一个 div 伪装为 td,display:table-row，再次嵌套一个 div 伪装成 td 元素：display:table-cell
  5. inline-block 大法
    父元素固定高度，子元素有宽度，无高度，display:inline-block,vertical-alian:middle。

子元素前后皆加上伪元素，伪元素高度 100%，display:inline-block,vertical-alian:middle。

Vue 中，$nextTick 是什么，一般用于什么场景？

Vue 中的 nextTick 涉及到 Vue 中 DOM 的异步更新。
在 Vue 生命周期的 created()钩子函数进行的 DOM 操作一定要放在 Vue.nextTick()的回调函数中。
在 created()钩子函数执行的时候 DOM 其实并未进行任何渲染，而此时进行 DOM 操作无异于徒劳，所以此处一定要将 DOM 操作的 js 代码放进 Vue.nextTick()的回调函数中。与之对应的就是 mounted()钩子函数，因为该钩子函数执行时所有的 DOM 挂载和渲染都已完成，此时在该钩子函数中进行任何 DOM 操作都不会有问题 。
在数据变化后要执行的某个操作，而这个操作需要使用随数据改变而改变的 DOM 结构的时候，这个操作都应该放进 Vue.nextTick()的回调函数中。
