CSS 中有哪些可以继承的属性

无继承属性：
可以记成“不论何，定背文”，不论怎么样，也要背文章。
意思是：

不（布局）：display

论（轮廓）：outline

何（盒子）： width,height,margin,padding,border 等

定（定位）：float,clear,position,top,right,left,bottom,overflow,clip,z-index 等

背（背景）：background

文（文本）：vertical-align,text-shadow,text-decoration,white-space

可继承属性：
可以记为：“字文表列，光可见”
字（字体）：font
文（文本）：text-align,line-height,color 等
表（表格）：border-collapse,border-spacing,empty-cells 等
列（列表）：list-style
光（光标）：cursor
可见（可见性）：visibility

css reset 和 normalize.css 的区别

因为各大浏览器厂商，没有一个统一的标准，所以它们之间会对 HTML 标签的默认样式渲染不同，比如有的 body 会默认有 8px 的外边距。
因此为了能够兼容各个浏览器，使页面的样式统一，这才有了样式重置（css reset）
它的目的就是统一不同浏览器之间的默认样式。

后面不断发展，有了相对比较温和的方案，就是 normalize.css
它保留了默认样式中有用的部分，同时保证了跨浏览器的一致性。

什么是 BFC？

BFC 翻译过来是“块级格式化上下文”，是一个独立的渲染区域。
里面的元素不超到外面来，而且外面的也不会影响这个矩形里的东西

哪些元素会生成 BFC?

根元素
float 属性不为 none
position 为 absolute 或 fixed
display 为 inline-block, table-cell, table-caption, flex, inline-flex
overflow 不为 visible
zoom: 1;

作用就是创立一个独立的块级格式化上下文

call，apply，bind 的区别

这三个方法的作用都是改变函数的执行上下文，也就是 this。
call,apply 通过设置函数内部的 this 和函数的参数来调用函数并返回函数的返回结果。
bind 把函数的 this 及部分参数固定为某个值，返回一个新的函数，调用新的函数的时候可以继续传入参数，最终已固定的参数和继续传入的参数会一起传给原函数，并把原函数的返回值返回。

call(),apply()的第一个参数相同，就是指定的对象。这个对象就是该函数的执行上下文，也就是 this。
call()在第一个参数之后的后续参数都是传入该函数的值，而 apply 只有两个参数，第一个是 this，第二个是一个数组。

如何判断一个对象是否为数组

法一：
Array.isArray(arg)

法二：
Object.prototype.toString.call(arg) === ‘[object Array]’

类数组如何转换为数组

法一：
Array.from(类数组)

法二： 把类数组当做是 Array.prototype 上的方法的 this 即可
Array.prototype.slice.call(类数组)
Array.prototype.concat.call(类数组)
或者：
[].slice.call(类数组)
[].concat.call(类数组)
